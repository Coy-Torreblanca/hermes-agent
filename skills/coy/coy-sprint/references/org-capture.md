# Org Capture — Hermes Create Todo

Append a `* TODO` entry to `/data/syncthing/Sync/org/inbox.org`.
Maker-mode only — no POINTS, VALUE, SPRINT, or GOAL. Triage handles routing and metadata later.
Matches Emacs `C-c c i` (Quick Inbox) — fast, minimal, no prompts.

## Canonical Source of Truth

The Emacs configuration at `/data/syncthing/Sync/org/emacs_config/` is the authoritative source for org-mode behavior. The markdown workflow files (`task_workflow.md`, `sprint_workflow.md`, `agenda_workflow.md`) are derivative and may be outdated. When in doubt, check the `.el` files under `emacs_config/lisp/`.

Key config evidence for captures:
- `common-org-config.el` lines 162-220: Capture templates use `* TODO` / `* STORY` at level 1, never `**`
- `common-org-config.el` lines 213-220: Quick Inbox (`C-c c i`) has ONLY `:ID:` and `:CREATED:` — no SPRINT, POINTS, VALUE, or GOAL
- `common-org-config.el` lines 264-302: Refile linter prompts for POINTS (all items), VALUE+GOAL+SPRINT (EPIC/STORY only) — these are triage-time, not capture-time

## When to Use

- Coy says "add a todo", "add to my list", "I need to remember to..."
- Coy fires off rapid items — capture each as its own TODO
- NOT for time-based reminders (those go to `reminder-db`)

## Capture Format (Matches Emacs `C-c c i`)

```
* TODO Title here
:PROPERTIES:
:ID:       <auto-generated UUID>
:CREATED:  [2026-05-14 Thu]
:SPRINT:   backlog
:END:
Body text — enough context for Coy to understand during later triage.
Multi-line body is fine. Keep it flat (no sub-headings in capture mode).
```

Key differences from task-with-properties captures (`C-c c t`):
- **No `:POINTS:`** — set during refile, not capture
- **No `:VALUE:`** — orthogonal to priority; prompted during refile for EPIC/STORY
- **No `:GOAL:`** — refile-only, never auto-derived from title

## Rules

1. **Append to end of file** — all captures go to one inbox. Triage decides destination.
2. **Body context required** — bare headline without body is useless in triage.
3. **Deterministic properties** — every capture includes `:ID:` (UUID), `:CREATED:` (today), `:SPRINT: backlog`. No POINTS, VALUE, or GOAL in inbox mode.
4. **Don't read inbox.org first** — just append. This is capture, not review.
5. **Fixed property order:** ID → CREATED → SPRINT → POINTS → VALUE → GOAL → TYPE → BRANCH → REPO. Every time. Never vary the order.
6. **Code change properties (TYPE/BRANCH/REPO):** Only include when appropriate. TYPE auto-generates BRANCH as `type/slugged-title`. REPO is optional.

## Implementation

### Primary: use `org_query.py --create-todo` (LIVE)

The `org_query.py` script is at `~/.hermes/scripts/org_query.py`. This is the canonical way to create todos. It handles UUID generation, timestamps, property ordering, and destination routing.

```bash
# Simple capture (title + body only — auto-defaults metadata)
python3 /data/.hermes/scripts/org_query.py \
  --create-todo '{"title": "Set up xurl CLI", "body": "Install the xurl CLI at the hermes skill path"}'

# With priority and tags
python3 /data/.hermes/scripts/org_query.py \
  --create-todo '{"title": "Fix login bug", "body": "Users can'\''t login after deploy", "priority": "A", "tags": ["bug", "urgent"]}'

# Direct to an EPIC in tasks.org (for pre-triaged items)
python3 /data/.hermes/scripts/org_query.py \
  --create-todo '{"title": "Add rate limiting", "body": "Prevent abuse on auth endpoint", "destination": "Personal AI v1", "points": 5, "value": "High", "keyword": "STORY"}'

# Preview without writing (dry-run)
python3 /data/.hermes/scripts/org_query.py \
  --dry-run '{"title": "Refactor auth middleware", "body": "Extract JWT logic into separate module"}'

# Code change story with TYPE/BRANCH/REPO (matches Emacs "d" template)
python3 /data/.hermes/scripts/org_query.py \
  --create-todo '{"title": "Add login page", "type": "feature", "repo": "github.com/Coy-Torreblanca/hermes-agent", "keyword": "STORY", "destination": "Personal AI v1", "points": 5}'
# Auto-generates BRANCH: feature/add-login-page

# Code change with explicit branch override
python3 /data/.hermes/scripts/org_query.py \
  --create-todo '{"title": "Fix auth bug", "type": "bugfix", "branch": "bugfix/auth-null-pointer", "repo": "github.com/org/repo", "keyword": "STORY"}'
```

### Fallback: `cat` heredoc append (when script unavailable)

```bash
cat >> /data/syncthing/Sync/org/inbox.org << 'EOF'

* TODO Set up xurl CLI
:PROPERTIES:
:ID:       A1B2C3D4E5F6G7H8
:CREATED:  [2026-05-14 Thu]
:SPRINT:   backlog
:END:
Install the xurl CLI at /data/.hermes/skills/social-media/xurl/
EOF
```

No reads. No patching. Single atomic append to end of file. The `cat << 'EOF'` pattern avoids printf char-escape issues entirely.

## Auto-generation Rules

- **ID**: `uuid.uuid4().hex[:16].upper()` — matching Emacs `org-id-get-create` format
- **CREATED**: System date in `[YYYY-MM-DD Day]` format
- **SPRINT**: Always `backlog` for inbox capture. Triage assigns the real sprint later.
- **VALUE**: NOT auto-derived from priority. Priorities and VALUE are orthogonal axes (different sort dimensions). VALUE is a triage-only property — include it in the block ONLY if the user explicitly provides it (e.g. `"value": "Essential"`). Emacs prompts for VALUE during refile on EPIC/STORY items, never during capture.
- **GOAL**: NOT auto-derived from title. GOAL is a triage-only property — include ONLY if explicitly provided. Emacs prompts for GOAL during refile on EPIC/STORY items, never during capture.
- **POINTS**: Omitted in inbox mode. Added during triage when refiled under an EPIC or STORY.
- **TYPE**: Code change type — "feature", "bugfix", "hotfix", or "chore". Only include if explicitly provided. When set, auto-generates BRANCH.
- **BRANCH**: Auto-generated as `type/slugged-title` when TYPE is provided (matches Emacs `my/format-branch-name`). Pass explicitly to override: `"branch": "bugfix/my-custom-name"`.
- **REPO**: Optional repo link. Can be used with or without TYPE. Example: `"repo": "github.com/owner/repo"`.

Canonical VALUE enum (from `common-org-config.el` line 226): `Essential`, `Important`, `Nice-to-have`. Legacy values (`Critical`, `High`, `Medium`, `Low`) also exist in older tasks but are not the canonical set.

## Implementation Notes

- `make_deterministic_block(params)` returns `(block_string, resolved_params)` — always unpack both.
- `--dry-run` returns the same `resolved_params` as `--create-todo`, including auto-derived `id`, `created`, `value`, `goal`. Never duplicate auto-derivation logic.
- The live script at `~/.hermes/scripts/org_query.py` is canonical. The skill copy at `/data/.hermes/skills/coy/coy-sprint/scripts/org_query.py` is synced from it.

## Pitfalls

- **Whitespace in titles** — if using heredoc, title whitespace is preserved. If using `--create-todo`, the JSON handles it.
- **Not for reminders** — time-based alerts use `reminder-db`. This is for task capture only.
- **Syncthing sync** — changes appear on the Mac within seconds. No manual sync needed.
